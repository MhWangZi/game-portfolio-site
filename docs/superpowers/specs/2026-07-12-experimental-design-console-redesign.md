# 实验控制台式游戏设计站点重构规格

日期：2026-07-12

## 1. 目标

将现有页面从纵向堆叠的游戏策划项目列表，重构为具有明确个人气质的“实验游戏设计控制台”。页面继续承担长期更新站点的职责，并通过全屏章节、真实项目图像、Three.js 状态联动和分层转场建立强识别体验。

访客进入后应在十秒内理解三件事：

1. 站点持续记录可玩原型、设计分析与工具项目。
2. 当前重点项目是 HD2DKit、Parry Arena 和 Anchored Gaze。
3. 每个项目都可以继续查看媒体、设计过程、下载包或外部链接。

本轮不修改隐藏后台认证方式、访问统计 Worker、下载文件位置和 GitHub Pages 部署方式。

## 2. 设计原则

- 实验性集中在滚动节奏、状态联动和项目切换，不依赖大量装饰文字。
- 每个屏幕只承担一个职责，减少 Hero、About、Archive、Notes 之间的重复说明。
- 真实游戏截图、宣传图和项目 UI 是主要视觉内容；系统分析条目只使用对应游戏的宣传图或实机图，不出现 Word 页面截图。
- 桌面端提供完整动效和空间层次，移动端保持连续阅读，不复制桌面端的复杂固定效果。
- 暗金琥珀是主页唯一主强调色。隐藏后台可继续使用绿色状态色，但不扩散到主页。
- 项目内容由数据驱动；组件不写死具体项目对象，也不重复维护项目文案与媒体路径。

## 3. 信息架构

页面由启动层和六个主要章节组成。桌面端章节默认使用 `min-height: 100svh` 和 CSS `scroll-snap-type: y proximity` 形成轻量章节吸附，不使用 JavaScript 强制锁定滚动；内容超过一屏时允许章节自然增高，不产生嵌套滚动。移动端关闭章节吸附，改为自然纵向流。

### 00 / Boot Sequence

- 持续约 1.4 秒。
- 中央的 Timing Gate 线框从四边组装，项目索引依次扫描。
- 状态文字控制在三到四行，进度使用细长线条。
- 动画结束后通过遮罩分离露出首屏，不进行突兀跳转。
- `prefers-reduced-motion` 下缩短为简洁状态切换，不播放线框组装和扫描。

### 01 / Current Builds

- 首屏使用 HD2DKit、Parry Arena、Anchored Gaze 三项目轮换。
- 默认项目为 HD2DKit；自动播放间隔 6 至 7 秒。
- 每个项目展示一个短标题、一句设计问题、三到四个标签、项目状态和一个主要入口。
- 图片、编号、状态面板和 Three.js 场景必须同步切换。
- 支持按钮、键盘方向键和触屏滑动；鼠标悬停时暂停自动播放。

### 02 / Design Radar

- 将“近期设计命题”和原能力矩阵压缩为一张交互节点图。
- 核心节点为原型验证、系统结构、行为观察、工具链整理。
- 悬停或聚焦节点时，中央图形、短说明、关键词与关联项目同步变化。
- 原能力矩阵中的七组数据作为关联项目与关键词来源，不再渲染七张长卡片。

### 03 / Case Files

- 使用整屏非对称档案轮播，保留 Parry Arena、Anchored Gaze、HD2DKit、STATIC SIGNAL 四个主案例。
- 左侧为真实图片主舞台，前后案例以半透明错位切片露出。
- 右侧为案例编号、一个设计问题、标签和操作入口。
- 底部使用时间轴式标签，不使用圆点分页器。
- 切换时使用图片切片滑出、编号闪烁、文案错层进入和 Three.js 状态变形。

### 04 / Project Index

- 删除当前十张长卡片组成的三列卡片墙。
- 桌面端改为左侧项目命令列表、右侧大型实时预览。
- 列表每项仅保留编号、标题、类型、一个设计问题和少量标签。
- 悬停或键盘聚焦列表项时更新右侧图片与状态；点击后打开项目档案面板。
- 继续支持全部、可玩原型、系统分析、工具项目等筛选。
- 移动端使用紧凑单列卡片，每个项目仍可直接打开详情。

### 05 / Field Notes

- 合并现有设计笔记与原型方法，避免两个区块重复表达设计过程。
- 上半部分为四条可展开的终端日志，结构为问题、观察、调整、结果。
- 下半部分为观察、拆解、假设、原型、复盘的流程轨道。
- 默认状态保持短文本；展开后显示完整内容，避免一次性填满页面。

### 06 / Contact

- 保留邮箱、GitHub、Bilibili 和抖音。
- 左侧为收束标题和当前状态，右侧为 2x2 联系入口。
- 增加返回顶部和最近项目入口，但不显示求职式能力说明。

## 4. 项目档案面板

原 `WorkDetail` 不再作为页面中 1500px 以上的常驻区块。点击项目后打开全屏档案面板，并继续使用项目 hash 作为深链。

档案面板包含以下频道：

- Overview：项目标题、类型、职责、引擎、时间和一句摘要。
- Media：该项目 `media` 数组中的全部图片或视频，可通过缩略轨道、键盘和触屏切换。
- Design：设计亮点、机制流程、执行内容和项目专属规则卡。
- Build：下载包、版本、大小、校验值和使用说明。
- Links：Bilibili、抖音、GitHub、GameJam 或其他外部入口。

档案面板支持 `Esc` 关闭、明确关闭按钮、焦点管理和浏览器前进后退。关闭后恢复打开前的章节与项目位置。移动端改为普通全屏纵向详情，不使用横向嵌套滚动。

## 5. 视觉系统

### Timing Gate

全站统一使用“外框 + 内部移动判定线”作为视觉符号。该符号来自弹反时机、地图层级和档案框架的共同结构，应用于：

- Boot Sequence 中央线框。
- 左侧章节轨道和当前编号。
- 按钮 hover / focus 状态。
- 项目轮换进度线。
- Three.js 主几何结构。
- 档案面板标题与媒体边框。

### 色彩与排版

- 主背景接近 `#05070b`，面板使用偏冷的深灰蓝。
- 暗金琥珀继续作为强调色，降低当前大面积暖色雾化背景。
- 中文标题使用清晰的系统无衬线字体栈；英文编号和状态信息使用等宽字体。
- 标题保持海报感，但不得裁切或强制形成不自然中文断行。
- 正文最多两至三行；长内容进入档案面板或展开日志。

### 固定界面层

- 顶部导航缩为窄状态栏，保留站点名与章节入口。
- 桌面端左侧显示 `01-06` 章节轨道，并随当前章节更新。
- 底部可显示低速状态文本带，但不得遮挡按钮或移动端内容。
- 所有固定层均设置明确的安全边距，并在窄屏关闭或收缩。

## 6. 动效系统

- 章节进入分别使用遮罩展开、文字错层、几何重组或媒体扫描，避免所有模块使用同一种淡入动画。
- 章节离开时只做轻微压暗和背景偏移，不阻止正常滚动。
- 使用 GSAP 管理章节时间线和 ScrollTrigger；CSS transition 负责 hover、focus 和短距离位移。
- Three.js 场景根据 `activeChapter`、`activeProjectId` 和 `motionMode` 接收状态，不能直接查询 DOM。
- 项目切换时 Three.js 改变几何结构、粒子速度和镜头目标，颜色仍保持单一暗金体系。
- 卡片 3D 倾斜限制在约 2 至 4 度，避免阅读区域明显变形。
- reduced-motion 模式停用自动播放、扫描、视差、复杂形变和大幅位移。

## 7. 组件与数据边界

`works.ts` 继续作为项目数据的唯一来源，并允许增加以下可选字段：

- `heroOrder`
- `shortTitle`
- `designQuestion`
- `archiveSummary`
- `scenePreset`

现有 `role`、`engine`、`period`、`summary`、`contribution`、`skills`、`media`、`download` 和 `links` 保持兼容。

推荐组件边界：

- `GlobalFrame`：顶部状态栏、章节轨道、底部状态文本。
- `CurrentBuilds`：首屏三项目轮换。
- `DesignRadar`：设计命题节点图。
- `CaseFiles`：主案例档案轮播。
- `ProjectIndex`：全部项目筛选与实时预览。
- `FieldNotes`：设计日志与方法轨道。
- `ProjectDossier`：项目全屏详情。
- `ContactSection`：联系方式和页面收束。
- `PortfolioScene`：只负责 Three.js 渲染，根据外部状态更新。

状态通过明确 props 和 hooks 传递：

- `useChapterController` 管理当前章节。
- `useProjectRotation` 管理自动播放、暂停和手动切换。
- `useDossierRoute` 管理项目 hash、面板开关和历史记录。
- `usePrefersReducedMotion` 继续提供动效降级。

隐藏后台 `PseudoAdmin` 和访问统计 `useVisitCounter` 保持独立，不依赖主页章节状态。

## 8. 数据与交互流程

1. `App` 读取 `works`，生成首屏项目、主案例、项目索引和设计雷达关联数据。
2. 章节控制器向 `GlobalFrame` 与 `PortfolioScene` 发布当前章节。
3. 项目轮换或列表选择向 `PortfolioScene` 发布当前项目，同时更新对应预览。
4. 点击项目后，路由 hook 写入项目 hash 并打开 `ProjectDossier`。
5. 关闭档案时恢复原章节位置；直接访问项目 hash 时自动打开对应档案。
6. `#/admin` 继续优先进入隐藏后台，不触发主页章节和访问计数逻辑。

## 9. 容错与降级

- 图片加载失败时按 `media` 顺序尝试下一张；全部失败时显示项目类型和编号组成的中性档案占位，不显示破图图标。
- 视频无法播放时显示 poster，并保留外部演示链接。
- 缺少下载或链接时隐藏对应操作，不保留空卡片。
- Three.js 初始化失败时页面内容和交互仍可使用，背景回退到静态网格与 Timing Gate 图形。
- 小屏和低性能设备使用精简粒子数量，不加载高成本后处理。

## 10. 响应式与可访问性

- 重点检查 1440x900、1280x720、390x844 三种尺寸。
- 1280x720 下首屏标题、状态面板和项目切换控件必须完整显示。
- 移动端取消左侧章节轨道、固定底栏、复杂 3D 倾斜和整屏滚动约束。
- 所有交互支持键盘焦点；轮播支持方向键，档案支持 `Esc`。
- 图片使用准确 alt 文本；纯装饰预览和 Three.js 画布对辅助技术隐藏。
- 焦点样式使用暗金描边，不能只依赖颜色表达选中状态。

## 11. 验收与测试

必须通过：

- `npm run build`
- `npm run lint`
- `npm run test:e2e`

浏览器验收覆盖：

- Boot Sequence 正常结束且不会阻塞阅读。
- 三项目首屏轮换、暂停、键盘和触屏切换可用。
- 六章节导航、当前章节轨道和滚动状态同步。
- Case Files 与 Project Index 使用真实项目图片。
- 项目档案深链、关闭、媒体切换、下载和外链可用。
- Three.js 画布非空，并能随章节或项目状态变化。
- 1280x720 无标题裁切、按钮溢出或大面积异常空白。
- 390x844 无横向溢出或固定层遮挡。
- reduced-motion 下内容完整且无自动播放。
- `#/admin`、密码门禁、访问统计和现有 Worker 地址不受影响。

## 12. 非目标

- 不在网页内直接运行 Windows EXE。
- 不新增内容管理、上传或真实服务端编辑功能。
- 不复制 JIEJOE 的源码、素材、字体组合或荧光绿视觉。
- 不为本轮重构生成伪装成真实游戏截图的 AI 图片。
- 不改变现有公开下载包和 Word 文档的文件格式。
