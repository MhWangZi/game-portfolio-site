# INDEX-0 内嵌线索与刷新重置修正规格

## 目标

修正公开页面谜题的两个体验问题：

1. 五段污染文字融入现有正文，不再以独立栏目、独立状态行或明显按钮出现。
2. 谜题进度只在当前页面生命周期内有效，浏览器刷新后全部清除。

## 状态规则

- `recoveredFragmentIds` 与 `phraseVerified` 只保存在 React 内存状态中。
- 页面内的 hash 路由切换不会卸载应用，因此从主页进入 `#/recovery`、再进入 `#/case-00` 时进度保留。
- 完整刷新会重新初始化为空状态：
  - 完整度恢复为 `0/5`。
  - 所有污染文字恢复为未发现状态。
  - 口令验证状态清除。
  - 已解锁的 `CASE-00` 状态清除。
- 未验证状态直接访问或刷新 `#/case-00` 时，现有路由守卫将用户送回 `#/recovery`。
- 不再为谜题状态读写 `localStorage`、`sessionStorage` 或 Cookie。

## 线索呈现

- `CorruptedFragment` 保留可聚焦的语义交互元素，确保键盘和触屏用户可以发现。
- 默认视觉与所在正文一致：无卡片、无底色、无独立边框、无常驻状态标签。
- 污染仅由替换字、竖线、下划线或轻微重影表达。
- 点击、悬停停留或键盘聚焦后恢复正确文字，并更新右下角完整度。
- 恢复后的文字继续留在原句中，不展开额外说明。
- `FRAGMENT XX / RECOVERED` 仅通过无障碍 live region 通知，不再作为可见标签占据版面。

## 五段正文位置

1. `被删除`：近期原型首屏的现有说明正文。
2. `的记录`：近期设计命题的现有说明正文。
3. `仍在`：设计档案当前条目的现有状态或说明正文。
4. `继续`：近期原型版本说明句，例如“下一轮将继｜续调整操作反馈与战斗节奏。”
5. `编写`：设计笔记或页脚的现有正文。

每个位置必须读起来像原本就存在的完整句子；移除专门解释 INDEX-0 异常的独立段落。

## 交互边界

- 桌面端：点击、键盘激活或短暂停留均可恢复。
- 触屏端：点击恢复，不依赖 hover。
- `prefers-reduced-motion`：直接切换到正确文本，不播放错位序列。
- 右下角 `ARCHIVE INTEGRITY` 保留，作为唯一常驻进度反馈。
- 五段全部恢复后，恢复入口仍按原逻辑出现。

## 验证

- 五段线索均位于正常文本流中，没有独立异常栏目。
- 第四段 `继｜续` 在近期原型页面可见、可聚焦、可点击。
- 找到部分线索后刷新，完整度回到 `0/5`。
- 完成口令并进入 `CASE-00` 后刷新，完成状态清除。
- 页面内完成五段、验证口令和进入隐藏档案的完整流程仍可运行。
- 桌面端、移动端、键盘操作和 reduced-motion 测试通过。
